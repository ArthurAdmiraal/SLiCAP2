Types

Core analyses#

SLiCAP's core analysis methods can help you obtain information of the following type:

  1. .transfer(): transfer functions
  2. .solve(): nodal voltages and branch currents
  3. .noise(): any information related to noise in the circuit
  4. .dcvar(): any information related to the effects of component or operating point variation
  5. .mna_matrix(): the MNA matrix representation of the circuit

You can enter an optional argument between brackets to specify the simulation type.

# defaultmy_circuit.transfer(sim='symbolic')
# if the numeric value of a symbol is known, it'll replace the symbolmy_circuit.transfer(sim='numeric')

.transfer()#

To obtain information about the transfer between 2 ports called port_in and port_out, use the following syntax:

tf = my_circuit.transfer().from('port_in').at('port_out')

The object tf contains an unevaluated transfer function. You can evaluate this transfer function by using the following methods:

tf.laplace # obtain the transfer function in the laplace domaintf.pz      # obtain the poles and zeroes of the transfertf.dc      # obtain the DC transfertf.impulse # obtain the impulse/step response after the transfertf.step    # obtain the impulse/step response after the transfer

.solve()#

SLiCAP can do nodal analysis. If you want to obtain a nodal voltage or a branch current, you can use the following syntax:

# obtain node voltage (w.r.t ground) of a node called 'out'result = my_circuit.solve().at('out')
# obtain branch current through a resistor 'R1'result = my_circuit.solve().through('R1')

The object result contains an unevaluated nodal analysis. You can evaluate this result by using the following methods:

result.laplace # obtain the transfer function in the laplace domainresult.dc      # obtain the DC transferresult.time    # obtain a time-domain expression

.noise()#

SLiCAP can find noise voltages and currents as well as noise transfers for you.

# find noise voltage at a node called 'out'result = my_circuit.noise().at('out')
# find noise current through a branch called 'R_L'result = my_circuit.noise().through('R_L')

You can refer output noise back to the input (or any other circuit node/branch).

# find noise voltage at the output (in this example: node 'out')# then, refer it back to the input (in this example: node 'in')result = my_circuit.noise().at('out').refer_to('in')

You can choose which noise sources to transfer to the output.

# find noise voltage at a node called 'out' as a result of the# noise source 'R1'.result = my_circuit.noise().from('R1').at('out')

The object result contains an unevaluated noise analysis. You can evaluate this result by using the following methods:

result.psd # obtain noise as noise PSD

.dcvar()#

Tolerances/variances/distributions of component values can change circuit behavior. SLiCAP can analyse the extend to which DC circuit behavior is affected.

Just like noise analysis, DC variation analysis considers the variance of component tolerances, and the transfer from that component to a node/branch.

Hence, the syntax of dcvar analysis is identical to that of noise, but instead of .noise() you type .dcvar().

Returned unevaluated analysis results do have different evaluations, though.

result.laplace # obtain the transfer function in the laplace domainresult.dc      # obtain the DC transferresult.time    # obtain a time-domain expression

.mna_matrix()#

You can obtain the Modified Nodal Analysis (MNA) Matrix of a circuit by using the .mna_matrix() method as follows:

# this commmand returns a sympy array with laplace variable smy_circuit.mna_matrix()

Detector types#

You use the detector type specifiers to specify the output quantity of the simulation.

.at(node)                # voltage at a node.into(terminal)          # current into a terminal.between(node_p, node_n) # differential voltage between two nodes (node_p - node_n).over(bipole)            # voltage over a bipole.through(bipole)         # current through a bipole

You can use lists to specify multiple detectors of the same type.

.at([node1, node2]).into([terminal1, terminal2]).between([node1_p, node2_p], [node1_n, node2_n]).over([bipole1, bipole2]).through([bipole1, bipole2])

Advanced Use#

.sample()#

Consider an analysis result that is a sympy expression in the time or s-domain. Using .sample() you can evaluate this function for a number of specified values.

# example: tf = 4*t + 2# .sample() works with listst_list = [1, 2, 3]tf.sample(t_list)>> tf = [6, 10, 14]
# .sample() works with numpy arraystf.sample(np.array([1, 2, 3]))>> tf = [6, 10, 14]
