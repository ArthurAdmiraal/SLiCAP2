Advanced use

Text-based schematic editing#

SLiCAP defines a circuit algebra that can be used for code-based manipulation of schematics. This is useful for automation, such as testbenches or group modifications.

caution

Once you give up on a visual schematic, it is easy to get lost in the code. Only use this functionality if you know what you are doing!

Components#

Circuits consist of components. Components are created by calling their respective class constructors, like all python objects.

resistor  = Resistor('R1')capacitor = Capacitor('C1')diode     = Diode('D1')mosfet    = NMOS('Q1')
info

Check the Components API Reference documentation for an exhaustive list of components.

Simple series-parallel circuits#

The most common need for text-based schematic creation is in building simple series-parallel networks for compensation. You use the + operator to make series connections.

resistor          = Resistor('R1')capacitor         = Capacitor('C1')series_rc_circuit = resistor + capacitor

Note that there is no need for an intermediate variable, the components can be defined directly in the series addition.

series_rc_circuit = Resistor('R1') + Capacitor('C1')

You use the | operator to make parallel connections.

parallel_rc_circuit = Resistor('R1') | Capacitor('C1')

The + and | parameters can be combined to make more complex circuits. Brackets are used to denote the order of operations.

compensation_circuit = Resistor('R2') + (Resistor('R1') | Capacitor('C1'))

Defining nodes#

Building series-parallel circuits has its uses, but is limited. What if you need to define an output node, for example in an RC filter? That can be done through adding named nodes. Nodes are defined through their class constructor, just like components.

node_out = Node("Out")

A named node can then be added using a + operator.

node_out = Node("Out")rc_filter = Resistor('R1') + node_out + Capacitor('C1')

As with components, no intermediate variables are required.

rc_filter = Node("In") + Resistor('R1') + Node("Out") + Capacitor('C1')

Ground connections can be defined in a similar way.

rc_filter = Node("In") + Resistor('R1') + Node("Out") + Capacitor('C1') + Ground()

Connecting terminals#

Not all circuits can be represented as series-parallel circuits. For example, a wye circuit cannot. For such cases, the terminal connection syntax can be used. In fact, this is the most versatile connection method. For complicated circuits, it is the method you use most often.

Terminals are connected using the + series connection operator.

R1 = Resistor('R1')R2 = Resistor('R2')R3 = Resistor('R3')
wye_circuit = R1.neg + R2.neg + R3.neg

As with series connections, named nodes can be added.

R1 = Resistor('R1')R2 = Resistor('R2')R3 = Resistor('R3')
wye_circuit  = R1.neg + R2.neg + R3.neg + Node('Center')

Component and terminal indexing is used to add more connections.

wye_circuit = wye_circuit.R1.pos + Node('In1')wye_circuit = wye_circuit.R2.pos + Node('In2')wye_circuit = wye_circuit.R2.pos + Node('In3')

Intermediate variables are not necessary. The same can be stated in a one-liner.

wye_circuit = (((R1.neg + R2.neg + R3.neg + Node('Center')).R1.pos + Node('In1')).R2.pos + Node('In2')).R2.pos + Node('In3')
caution

Even for this simple circuit, cascading connections on a single line quickly gets unreadable. Often, it is better to separate your connections over multiple lines. One case where single line connections by terminal may be useful is in explicitly specifing the direction of directed bipoles, such as diodes.

clamp_circuit = (Node('In') + Resistor('R1') + Node('Out') + Zener('D1').K).D1.A + Ground()

The terminal connection syntax can also be used to merge named nodes. The name of the node on the left of the + operator is the name of the new node.

source = Node("Out-") + VoltageSource('Vs') + Resistor("Rs") + Node("Out+")load   = Node("In+") + Resistor('Rl') + Node("In-")
circuit = source.nodes['Out+'] + load.nodes['In+']circuit = circuit.nodes['Out-'] + circuit.nodes['In-']# the only named nodes in the finished circuit are 'Out+' and 'Out-'

Combination with series connections#

Terminal connections can be combined with series connections. In fact, we have already used the terminal connection syntax to add nodes within series connections! When the + operator is used with only monopoles, a terminal connection is made. When it is used with at least one multi-pole circuit or component, a series connection is made. Let's look at an example.

# every term is a monopole - all of the terminals are connected in a single nodewye_circuit    = Resistor('R1').neg + Resistor('R2').neg + Resistor('R3').neg# the middle term is a bipole - the terminals are connected on either sideseries_circuit = Resistor('R1').neg + Resistor('R2') + Resistor('R3').neg

This is useful when adding networks around multipole components, such as an opamp.

amplifier = Opamp('U1').out + (Resistor('R1') | Capacitor('C1')) + Node('FB') + Resistor('R2')amplifier = amplifier.nodes.FB + amplifier.U1.in_n

Combination with parallel connections#

Parallel connections can also be combined with terminal connections, but are more tricky to work with. When series and terminal connections are made, the leftmost and rightmost node are tracked. The | paralell connection operator then connects these nodes. This gives the expected behaviour for series-parallel RLC circuits.

circuit = (Resistor('R1') + Inductor('L1') + Capacitor('C1')) | (Resistor('R2') + Capacitor('C2'))

The behaviour is more involved once multipole components are added to the circuit.

circuit = ((Resistor('R1') + NMOS('Q1').gate).Q1.drain + Resistor('R2')) | Resistor('R3')# results in a circuit where R1 is connected to the NMOS gate, R3 is connected to R1,# and R2 connects the other side of R3 to the NMOS drain

Coupling of component variables to circuits#

A component variable can be used to change its parameters in the circuit to which it is coupled. Most operators don't preserve this coupling though.

R1       = Resistor('R1')R1.value = 10circuit  = Ground() + VoltageSource('V1') + R1 + Ground()R1.value = 20print(circuit.R1.value) # returns 10 ohm

Without taking special care, all connection operators break the link between the component variable and the circuit. Hence, parameters need to be adjusted using indexing.

circuit.R1.value = 30print(circuit.R1.value) # returns 30 ohm

It is possible to create a new component variable that is linked to the circuit.

R1 = circuit.R1R1.value = 40print(circuit.R1.value) # returns 40 ohm

Preserving component variable couplings with the union operator#

The reason operators break coupling by default is that operators on circuits create new instances for all of the components in the circuit. This is useful because it prevents changing a component's parameters in one circuit from effecting another. However, there & union operator can be used to work around this if you have only one instance of a circuit.

Instead of copying all of the components, the & union operator combines all of the components, connections, and nodes in its operands. If two components or nodes have the same name, those from the older circuit are kept.

R1       = Resistor('R1')R1.value = 10circuit  = R1circuit &= Ground() + VoltageSource('V1') + R1 + Ground()R1.value = 20print(circuit.R1.value) # returns 20 ohm

This is useful for digital-like circuits where components have so many connections that using indexing is cumbersome.

entity1 = Entity('E1')entity2 = Entity('E2')
circuit  = entity1 & entity2circuit &= entity1.In0 + entity2.Out0circuit &= entity1.In1 + entity2.Out1circuit &= entity1.In2 + entity2.Out2circuit &= entity1.In3 + entity2.Out3

Removing components#

So far, we have discussed how to add to circuits. But sometimes, you need to remove connections to make space for something new. For example, if you want to replace a voltage divider by a transactor, you remove it.

Both components and nodes have a remove() method to remove it from the circuit, along with its connections.

V1 = VoltageSource('V1')R1 = Resistor('R1')R2 = Resistor('R2')circuit = Ground() + V1 + R1 + Node("Out") + R2 + Ground()circuit.R2.remove() # circuit now only has V1, node out and R1circuit.nodes.Out.remove() # circuit now only has V1 and R1

It is also possible to remove a list of objects directly from the circuit.

circuit.remove([circuit.nodes.Out, circuit.R2]) # circuit now only has V1 and R1

Terminals cannot be removed on their own, as it would alter the component. Instead, they can be disconnected from a node by the disconnect method.

V1 = VoltageSource('V1')R1 = Resistor('R1')R2 = Resistor('R2')circuit = ()(Ground() + V1 + R1.pos).R1.neg + R2.pos).R2.neg + Ground()circuit.R2.pos.disconnect() # circuit still has all components, but R2 is no longer connected to R1circuit.R1.pos.disconnect() # circuit still has all components, but R2 and R1 are no longer connected

As with remove, it is also possible to disconnect a list of terminals directly from the circuit.

V1 = VoltageSource('V1')R1 = Resistor('R1')R2 = Resistor('R2')circuit = ()(Ground() + V1 + R1.pos).R1.neg + R2.pos).R2.neg + Ground()circuit.disconnect([circuit.R2.pos, circuit.R1.pos])# circuit still has all components, but R2 and R1 are no longer connected

Combining subcircuits#

You combine circuits to reuse models in multiple analyses. For example, you combined a source model with a nullor-based amplifier concept to check the transfer parameters, and later with the small signal design of the amplifier to check the dynamics.

You can combine circuits using the standard connection operators.

# define source circuitsource = Node("Out-") + VoltageSource('Vs') + Resistor("Rs") + Node("Out+")
# define amplifier circuitamplifier = Nullor('N1').out_p + Resistor('R1') + Node('FB') + Resistor('R2')amplifier = amplifier.nodes.FB + amplifier.N1.in_namplifier = amplifier.N1.out_n + Ground()amplifier = amplifier.N1.out_p + Node("Out")amplifier = amplifier.N1.in_p + Node("In")
# combine circuits in testbenchtestbench = amplifier.nodes.In + source.nodes['Out+']testbench = testbench.nodes['Out-'] + Ground()
