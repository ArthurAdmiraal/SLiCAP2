Design principles

Reuse as much of the interface as possible#

Examples:

  • The 18 different analyses types have been consolidated in variations on 5 base analyses, mostly by making the output domain selectable. (learnability)
    • This has actually increased the amount of possible analyses. (flexibility)
  • The detector syntax is the same among all analyses. (learnability)

Optimisation shouldn't get in the way of the user#

Examples:

  • Rather than setting rigid conditions on the input, the similarity of parameter stepping analyses should be detected by SLiCAP.
    • This brings performance speedup in more cases. (flexibility)
    • This ensures maximum flexibility. (flexibility)

Only do tasks unique to SliCAP#

Examples:

  • Instead of defining a stepping syntax that is specific to SliCAP, parameter stepping is handled by python for loops.
    • Instead of learning something specific to SLiCAP, the user learns a general principle. (learnability)
    • This gives the user access to the full flexibility of python. (flexibility)

Documentation should be short, but comprehensive#

Examples:

  • Inspired by Docusaurus, sentences contain no fluff. (learnability)
  • Every bit of functionality has a tutorial with short usage cases. (learnability)
