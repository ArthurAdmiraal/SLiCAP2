Working with parameters

Parameters define every aspect of the circuit from its connectivity to its behaviour. As such, you access parameters any time you change aspects of the circuit or inspect properties of the circuit. For example, you change the value parameter of a Resistor to change the attenuation of a voltage divider. Each circuit object has its own set of parameters.

Component parameters#

The parameters that determine the behaviour of the circuit are component parameters. For example, the value of a resistor will determine the current that flows through it at a given voltage. As such, most of the parameters you will need to edit are component parameters.

Component parameters are accesed via dot indexing.

resistor.value # returns the resistance, e.g. 10resistor.value = 20
caution

Not only parameters are accessed via dot indexing. It is also a shorthand way to access terminals.

The types of component parameters are unique to each component, but some are shared by all components. These are the parameters that do not concern the behaviour of the component, but instead serve to identify the component. There are three such parameters that are shared by every component.

component.name      # e.g. 'R10'component.name = 'R11'component.terminals # e.g. [Terminal('pos'), Terminal('neg')], cannot be modified
info

Check the Components API Reference documentation for an exhaustive list of component properties.

Though the behavioural parameters are unique to each component, some are shared by all components of a class. We will now have a look at the parameters of the most common classes.

Common component parameters#

Bipoles#

Every Resistor, Capacitor, and Inductor is a Bipole. These all have the value parameter.

resistor.value  = 10   # ohmcapacitor.value = 1e-6 # Finductor.value  = 1e-6 # H

Transactors#

Every VoltageControlledVoltageSource, VoltageControlledCurrentSource, CurrentControlledVoltageSource, and CurrentControlledCurrentSource is a Transactor. These all have the value parameter.

vcvs.value = 1e6  # voltage gainvccs.value = 10   # transimpedanceccvs.value =  0.1 # transadmittancecccs.value =  2   # current gain

Node parameters#

Node parameters serve only to identify nodes and define connectivity. For example, you refer to a terminals by its name. The types are common to all nodes and they are accesed via dot indexing.

node.name      # e.g. "Out"node.name = "FilterOut"node.terminals # e.g. [R1.neg, C1.pos], cannot be directly modified
info

Check the Nodes API Reference documentation for an exhaustive list of node properties.

caution

Naming nodes using parameters is not a replacement for properly named schematics. It is easy to lose track of what's going on once you abandon schematics.

Terminal parameters#

Terminal parameters serve only to identify terminals and define connectivity. They cannot be directly modified. For example, you refer to a terminal by its name. The types are common to all nodes and they are accesed via dot indexing.

terminal.name # e.g. "neg", cannot be modifedterminal.node # e.g. Node("Out"), cannot be directly modified
info

Check the Nodes API Reference documentation for an exhaustive list of terminal properties.
